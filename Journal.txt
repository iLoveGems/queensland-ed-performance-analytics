2/06/2026:
Entry 1 - Data Acquisition
	
	> Downloaded historical Emergency Department datasets from the Queensland Open Data Portal.
	> Identified that historical files were supplied in multiple formats (.xls and .csv)
	> Converted all files to .csv format simplify ingestion and standardisation.


------------------------------------------------------------
Entry 2 - Initial Data Investigation

	> Obtained the accompanying Queensland Health data dictionary describing each variable.
	> Imported the documented column descriptions into the master dataset.
	> During validation, discovered that the documented column order did not align with the actual data contained in historical files.
	> Determined that applying the documented column names directly resulted in incorrect interpretation of several performance measures.


------------------------------------------------------------
Entry 3 - Schema Validation

	> Compared multiple historical datasets (2015, 2018, 2019) against the current 2025 quarterly dataset.
	> Identified that historical datasets used a different column ordering for several Emergency Department performance measures.
	> Investigated the relationship between waiting times metrics, admission metrics, and treatment completion metrics to determine the correct field mappings.
	> Developed a standardised column mapping to align historical data with the current Queensland Health reporting schema.


------------------------------------------------------------
Entry 4 - Data Transformation

	> Created a Power Query workflow to import and combine historical datasets.
	> Applied standardised column names across all historical files.
	> Corrected column mappings to ensure performance measures aligned with the correct fields.
	> Validated transformed data against original sources files to confirm accuracy.


------------------------------------------------------------
Entry 5 - Data Quality Assurance

	> Conducted row-level validation using August 2015 source data.
	> Verified attendance counts, waiting time measures, admission percentages, and treatment completion metrics against the original dataset.
	> Confirmed that transformed records matched source records with no identified discrepancies.


------------------------------------------------------------
Entry 6 - 

	> The 2022 quarterly dataset contained a text placeholder in attendance variation field rather than a numeric value.
	> This value was treated as missing (0) during cleaning to preserve numeric data integrity


------------------------------------------------------------
Entry 7 -

	> Datasets were separated into three .csv files: 2015-2022, 2023-2024, 2025. This was done due to inconsistencies with column 	names and orders, removal of an additional column in 2023-2024. The columns were standardised to match the latest 2025 format.
	> All were merged into one master file.
	> Final validation identifies no duplicate valid record, no invalid percentage ranges, and no attendance count inconsistencies between triage-level rows and ALL rows.
	> Six blank system-generated rows with a 0/01/1900 date were removed.
	> A negative median waiting time value was treated as a source-data anomaly and set to null prior to analysis.


 ------------------------------------------------------------
Entry 8 - 

	> Facility naming conventions varied across reporting periods despite sharing the same facility code.
	> A standard facility naming convention was developed and applied using Power Query to ensure consistency across historical and current datasets.


------------------------------------------------------------
Entry 9 - Creation of Analytical Dataset

	> Following completion of data cleaning and validation, all datasets were consolidated into a single master Emergency Department dataset.
	> The final dataset contains records from July 2015 to September 2025.
	> Additional temporal variables (Month, Quarter, Year) were created to support trend analysis.
	> The final dataset was exported for loading into PostgreSQL and subsequent analysis using SQL and Power BI.


------------------------------------------------------------
Entry 10 - Initial SQL Validation

	> Imported the final master dataset into PostgreSQL.
